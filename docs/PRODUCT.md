# Product

## Register

product

## Users

Developers who install SAG to automate open-source project setup. Two postures:
launching setups (single or batch) and walking away, then returning to check
progress, diagnose failures, and read setup reports. Often first-time users;
the UI must explain itself without a manual. Runs locally next to a terminal
and is frequently glanced at rather than lived in.

## Product Purpose

SAG Workbench is the local control surface for SAG's LLM-driven setup agents:
launch project setups (single or batch), watch container and agent state,
inspect session evidence (context maps, logs, reports), and intervene when a
setup fails. Success: a user can tell at a glance which setups need attention,
and can find the "why" for any failure within two clicks.

## Brand Personality

Calm mission-control. Quiet confidence: dense but legible information,
restrained color where status alone carries hue, nothing animates or colors
itself without meaning. Attention is a budget spent on failures and running
work. Three words: observant, precise, unhurried.

## Anti-references

- Generic SaaS analytics dashboards: hero metrics, identical card grids,
  gradient accents, decorative illustration.
- Anything that hides engineering detail behind gloss; ids, refs, paths, and
  exit codes are first-class content here, not noise to be tucked away.

## Design Principles

1. Status earns color: hue belongs to state (running, failed, needs
   attention), never to decoration.
2. Glanceability first: the dashboard answers "is anything wrong?" before any
   other question.
3. Evidence within two clicks: every state shown links to the artifact that
   proves it (log, report, context map).
4. Density with hierarchy: many rows are fine; scale and weight contrast make
   scanning effortless.
5. Self-explanatory on first run: empty states and affordances teach; no
   manual required.

## Accessibility & Inclusion

WCAG AA basics: body text contrast at or above 4.5:1, keyboard operability for
all actions, visible focus states, and prefers-reduced-motion alternatives for
every animation.
